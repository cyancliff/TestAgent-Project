## 摘要

多模态人格分析旨在通过融合相关的多模态信息来准确检测人格特质。然而，现有方法大多聚焦于单模态特征，而忽视了对于这一跨学科任务至关重要的双模态关联特征。为此，我们提出了一种基于自适应图 Transformer 网络和多任务学习的多模态人格预测框架。首先，我们利用预训练模型学习不同模态的特定表示。具体来说，我们将预训练多模态模型中的编码器作为模态专属特征提取方法的骨干网络，用于挖掘单模态特征。进一步地，我们引入一种新的自适应图 Transformer 网络来挖掘与人格相关的双模态关联特征。该网络能够基于关系图有效学习高阶时间依赖，并强调更重要的特征。此外，我们使用一种多模态通道注意力残差融合模块获取融合特征，并提出一种多模态与单模态联合学习的回归头，用于学习并预测人格特质得分。我们还设计了一个多任务损失函数，以增强人格预测的鲁棒性和准确性。在两个基准数据集上的实验结果表明，我们的框架是有效的，并优于现有最先进方法。代码地址为：<https://github.com/RongquanWang/PPF-AGTNMTL>。

## 1. 引言

人格计算是情感计算的一个分支，研究人格特质的分析、识别与合成。人格特质是稳定的思维、情感与行为模式，它们塑造并区分了个体的人格 [RK23]。一些研究者使用心理学量表建立了不同的人格特质模型。例如，大五人格模型通过五个特征来评估人格：开放性（Open.）、尽责性（Cons.）、外向性（Extr.）、宜人性（Agre.）和神经质（Neur.）。随着 YouTube、Instagram、抖音等多媒体社交平台的兴起，人们越来越频繁地通过视频分享生活 [ZWL∗23]。与此同时，CVPR 在 2016 和 2017 年举办了基于 YouTube 视频的人格预测竞赛 [GGB∗17]，并发布了 First Impressions v2 数据集 [EKS∗18]。这些视频都带有基于人格特质的外显人格分数标注。基于这些事实，从短视频中自动推断个体的大五人格成为可能。

人格特质会显著影响我们的生活，并作用于行为、认知、心理健康、偏好、欲望和决策等方面。此外，自动检测人格特质具有重要的实际应用价值，例如求职面试 [KGAS17]、推荐系统、社交网络分析、个人助理、购买行为分析、医疗健康与咨询、犯罪学等。例如，在企业招聘新员工时，可以基于面试者的第一印象自动识别人格特征，从而筛选出更合适的候选人。

多模态人格分析旨在研究多模态数据中所表达的人格特质 [LLT22]。由于基于多模态信息的方法能够提升预测的鲁棒性 [SUS14]，大量多模态人格预测方法被提出。随着对不同模态数据进行组合、筛选和特征提取，相关方法不断涌现 [LSG24]。然而，模态之间天然存在异质性，这往往会引入信息冗余和分布差异，从而增加多模态表示学习和特征融合的难度。

首先，一些仅基于视觉信息的方法通过卷积神经网络或 3D CNN 结构提取视觉特征，并用于人格预测，如 [EBDOTA17]、[HK20]、[WSC∗20]、[HSS18]。这些方法主要关注视觉特征，忽略了音频和文本信息，对多模态数据的利用有限。因此，基于视觉与音频信息的方法通过双流网络提取视觉中的人脸和场景特征以及音频特征来进行人格预测。这类方法目前是主流人格预测方法，包括 [GGvGvL16]、[SPM∗16]、[KGAS17]、[VML17]、[PPJE19]、[SHZ∗22]、[SHL20]、[MAMCMS20]。与此同时，也有一些面向其他任务的音视频理解方法被用于提取视听特征并训练人格预测回归模型，例如 [LSL∗23] 聚焦于预训练 ViT 与交叉注意力机制，[GCDZ19] 利用人体手部与面部动作特征，[TLX20] 使用单模态与跨模态时间上下文，[HLTH24] 是长时音视频理解方法，[XZ22] 引入跨模态门控注意力机制，[GCX23] 提出了联合模态互学习（JML）过程，用于自适应动态校准多种音频和视觉信息。

考虑到文本模态同样有助于提升人格预测精度，一些同时基于视觉、音频和文本信息的方法也被提出，它们通过不同架构整合多模态特征，借助空间模型、时间模型和注意力机制提取并融合信息，例如 [ZLT∗23]、[RK23] 和 [GGB∗17]。具体来说，这些方法尝试过多种融合策略，例如 [WZZW17] 与 [ZZWW16] 的早期融合和晚期融合，[SSG∗22] 通过实验发现直接拼接多模态特征效果更好，[WLFO23] 设计了情绪引导的多模态融合与对比学习框架。与此同时，也有研究提出新的损失函数以缓解模型训练不准确问题，如 [LWM∗20] 的 Bell Loss 和 [BLQ∗24] 的 Gaussian Loss。此外，一些方法还关注人格预测模型的可解释性，例如 [EKS∗18] 和 [EKS∗20]。

近年来，预训练在多模态处理领域取得了快速发展 [AYQ∗21]，尤其是多模态预训练模型 [RLS∗23]，例如 CLIP [RKH∗21] 以及从 CLIP 蒸馏得到鲁棒音频表示的 Wav2CLIP [WSKB22]。这些模型不仅能够在不进行微调或不设计任务特定架构的情况下对不同领域和任务表现出良好的泛化能力，还可以提供与类别标签对应的嵌入，因此非常适合作为小样本人格预测任务的特征提取器。例如，基于 CLIP 的可解释人格预测模型 [GSM23] 已经被提出。进一步地，[SSJ∗21] 搜索最优 CNN 架构，并据此构建了人类特异的图结构表示来识别被试人格。这种图结构表示提出了一种新的端到端边特征学习策略，有助于保留人类特异 CNN 的独特参数及其层间几何关系。

尽管已有许多方法被提出，但它们大多提取并融合单模态特征，而忽视了多模态关联特征；同时，它们尚未充分利用图结构去探索不同帧特征之间的内在关系信息，也未能充分分析多模态融合方法的有效性 [RLDL24]。此外，现有人格预测方法大多基于单任务学习，因此无法更好地挖掘不同任务之间的共享信息和互补特征 [LS23, YXYW21]。与单任务学习相比，多任务学习能够提升泛化能力、降低模型过拟合风险，从而增强训练模型的鲁棒性。不同于以往工作主要关注多任务间的权重调整，我们的方法通过多任务损失共同学习单模态任务和多模态任务。它既能从多模态任务中学习相似信息，也能从单模态任务中学习差异化信息。通过联合训练多模态任务和单模态子任务，我们的框架能够更多关注每个单模态自身的差异化表示。

针对以上挑战，我们提出了一种基于预训练模型和自适应图 Transformer 网络的新型多模态人格分析框架。该框架将一种单模态特征与两种双模态关联特征纳入多任务联合学习过程，从而同时建模模态间的时空特征。我们利用训练好的 CLIP 文本编码器、双向门控循环单元（BiGRU）[HZXL21]、时间注意力块（TAB）以及平均操作，从音频转写文本中挖掘文本特征，形成单模态特征。我们利用 CLIP 与 Wav2CLIP 提取文本、视觉和音频的空间特征，用于构建双模态关联特征。随后，通过向量积和本文提出的自适应图 Transformer 网络来学习单模态序列特征中的更长时间依赖。最后，利用余弦相似度和平均操作提取场景-描述关联特征与场景-音频关联特征。我们首先采用多模态通道注意力残差融合模块对多模态任务进行特征融合，最后设计多任务联合学习回归头和多任务损失，以训练并预测人格特质得分。

我们的主要贡献总结如下：

- 我们提出了一种新的多模态人格预测框架，采用自适应图 Transformer 网络提取两种双模态关联特征，能够有效捕捉非语言信息中模态内动态的时间依赖。
- 我们设计了一个多任务联合学习回归头，以更有效地学习与人格相关的线索；进一步地，我们定义了新的多任务损失函数，以增强人格预测的鲁棒性和准确性。
- 在两个基准数据集上的大量实验验证了该框架的有效性；同时，我们的方法取得了最先进性能。

本节将详细介绍我们提出的方法，其整体框图如图 1 所示。框架输入为一段短视频，主要由特征提取模块、多模态通道注意力残差融合模块、多任务联合学习回归头模块以及多任务损失模块组成。下面分节介绍。

> **图 1：** 我们框架的整体结构。框架首先使用 CLIP 的文本编码器和 Wav2CLIP 提取多模态空间特征；随后使用 MLP 和本文提出的自适应图 Transformer 网络学习不同单模态中的时间关系，并通过余弦相似度从视觉和音频信息中获取两类关联特征；然后使用多模态通道注意力残差融合模块进行特征融合；最后使用多任务联合学习回归头预测人格特质分数。

### 2.1 特征提取模块

本文使用 CLIP 和 Wav2CLIP 作为特征提取器。我们利用它们的视觉编码器为场景图像生成嵌入，利用其文本编码器或音频编码器生成文本或音频线索嵌入。

#### 2.1.1 基于预训练 CLIP 的场景-描述关联特征

我们设计了一种基于 CLIP 的场景-描述关联特征提取方法 [GSM23]。该方法包含三个主要步骤：

1. 使用 Wav2CLIP 的视觉编码器为视觉场景线索生成嵌入；
2. 使用 CLIP 的文本编码器为人格描述生成嵌入；
3. 通过对场景嵌入和描述嵌入做向量积，计算二者之间的相似度分数，并通过平均操作与 MLP 降低特征维度。

本文设置 `N = 15`，`N' = 1608`。我们使用一个人格描述词列表来生成描述嵌入，其中包含 `1608` 个能够准确描述人格的形容词。该列表由人格调查中的术语 [CM08, Joh91] 与多部词典 [Dic89, Dic02, Mat09] 中精选而来，以保证描述的多样性与代表性，从而有效捕捉人格复杂性。随后，我们使用 CLIP 的文本编码器为这些人格描述生成描述嵌入。同时，我们使用基于 ViT 架构的 `openai/clip-vit-large-patch14-336` 模型作为 Wav2CLIP 的视觉编码器，为视频中提取的场景图像生成嵌入。具体而言，设 `P = {p1, p2, ..., pN'}` 表示 `N'` 个人格描述组成的集合，`V = {v1, v2, ..., vN}` 表示从一个视频中提取的 `N` 张场景图像。对所有人格描述 `P`，我们用 CLIP 文本编码器生成嵌入 `TP`；对所有场景图像 `V`，我们用 Wav2CLIP 视觉编码器生成嵌入 `SV`。

利用场景图像嵌入 `SV ∈ R^{N×768}` 和人格描述嵌入 `TP ∈ R^{N'×768}`，我们通过向量积计算两者之间的相似度分数，并将其作为特征向量，进而得到特征矩阵 `F ∈ R^{N×N'}`：

```text
Fi,j = SVi^T TPj^T    (1)
```

其中，`Fi,j` 表示第 `i` 张场景图像与第 `j` 个人格描述之间的相似度分数。这样一来，每一张场景图像都得到一个长度为 `N'` 的特征向量，其中每个值对应于该图像帧与相应人格描述之间的相似度。

最终，我们能够学习到一个场景图像与人格描述的联合嵌入空间，使得场景图像及其相关描述被映射到同一空间。然后，对每个视频中的所有场景图像特征向量求平均，得到场景-描述关联特征 `Fsd ∈ R^{1×N'}`：

```text
Fsd^j = (1/N) * Σ(i=1→N) Fi,j    (2)
```

其中，`Fsd^j` 是该视频针对第 `j` 个人格描述的特征值，`N` 为视频中的帧数。

我们构建并训练了一个两层全连接的 MLP 回归模型来降低场景-描述关联特征 `Fsd` 的维度。每个全连接层由线性层、BN 层、ReLU 激活层和 Dropout 层（参数 `p=0.3`）组成，将特征维度从 `1608` 依次降到 `512`，再降到 `128`。最终得到场景-描述关联特征 `fsd ∈ R^{1×128}`。

#### 2.1.2 基于预训练 Wav2CLIP 的场景-音频关联特征

如图 2 所示，为提取场景-音频关联特征，我们首先构建场景图像关系子图和音频 patch 关系子图，然后提出自适应图 Transformer 网络分别聚合单模态音频和场景特征，最后利用相似度分数得到场景-音频关联特征。下面给出具体过程。

##### 2.1.2.1 场景图像关系子图

基于已获得的场景图像特征矩阵 `SV ∈ R^{N×768}`，我们先使用正弦和余弦函数为每个场景图像特征生成位置编码向量，并将其与原始场景图像特征相加，得到 `V'`。为了构建场景图像关系子图 `GV = {CV, EV}`，其中节点 `CV` 用于捕捉不同帧场景图像的语义，边 `EV` 用于度量不同节点（帧）之间的关系。这里，视觉特征 `V'` 被作为场景图像关系子图的节点 `CV = {ciV}^N_{i=1} ∈ R^{N×dV}`，其中 `ciV` 和 `dV` 分别表示第 `i` 帧场景图像的视觉特征及其维度。本文中，`GV` 中所有节点都两两相连，即它是一个完全图。

##### 2.1.2.2 音频 patch 关系子图

本文首先使用 `ffmpeg` 工具将视频转换为音频数据，然后将音频转换为 `log-mel spectrogram`。接着，我们将该谱图划分为若干 patch，并将每个 patch 单独输入 Wav2CLIP 的音频编码器提取 patch 特征，从而得到 patch 特征矩阵 `A' ∈ R^{N×512}`，其中 `N` 为该视频中的 patch 数量。在不失一般性的情况下，音频 patch 关系子图 `GA` 的构建方法与场景图像关系子图相同。

##### 2.1.2.3 自适应图 Transformer 网络

本文提出的自适应图 Transformer 网络由两个连续的图 Transformer 模块组成。每个图 Transformer 模块包括三部分：位置编码器、图卷积网络（GCN）卷积模块和 Transformer 编码器。

首先，我们使用位置编码器为特征生成位置向量 `PE`，并将其加到原始特征上。由于数据序列长度固定，本文采用 Transformer [VSP∗17] 中的绝对位置编码器，使用正弦和余弦函数在不同位置生成特征向量。

其次，我们使用一层标准 GCN 卷积提取空间特征。GCN 卷积可表示为：

```text
X' = D^(-1/2) Â D^(-1/2) XW + b    (3)
```

其中，`X` 是场景图像特征矩阵或音频 patch 特征矩阵的输入，`X'` 是 GCN 的输出，`Â` 是带自环的邻接矩阵，`D` 是 `Â` 的度矩阵，并满足 `Dii = Σj Âij`。`W` 和 `b` 分别是 GCN 卷积的权重和偏置。经过 GCN 卷积之后，本文使用 BatchNorm 正则化，加入残差结构，并最终使用 ReLU 激活函数 `σ`。这一过程表示为：

```text
fGCN = σ(X + BN(X'))    (4)
```

接着，我们使用一层标准 Transformer 编码器层来计算时空特征。自注意力机制可表示为：

```text
Q, K, V = fGCN WQ, fGCN WK, fGCN WV    (5)
Attention(fGCN) = softmax(QK^T / √dk) V    (6)
```

其中，`Q`、`K`、`V` 分别为 Query、Key 和 Value 矩阵，`WQ`、`WK`、`WV` 为对应的可学习参数，`dk` 表示 `Q` 和 `K` 的特征维度。多头注意力机制表示为：

```text
MultiHead(fGCN) = Concat(head1, ..., headi, ..., headh)    (7)
headi = Attention(fGCN), i ∈ [1, ..., h], h = 8           (8)
```

其中，`MultiHead(fGCN)` 是 Transformer 的多头自注意力模块。本文将注意力头数设为 `h = 8`，并使用 `dropout = 0.1`。

最后，在通过自注意力模块输出输入图的结构特征后，本文继续采用残差结构，并使用 LayerNorm（LN）做正则化，再通过两层全连接层进一步学习特征。该过程表示为：

```text
fMA = LN(fGCN + MultiHead(fGCN))                    (9)
fgraph = LN(fMA + (σ(fMA W1 + b1)) W2 + b2)        (10)
```

其中，`W*`、`b*` 是两层全连接层的权重和偏置。

##### 2.1.2.4 场景-音频关联特征

在构建好场景图像关系子图 `GV` 和音频 patch 关系子图 `GA` 后，我们首先对这两个关系图进行最大池化操作，分别得到场景图像关系子图聚合特征 `fV` 和音频 patch 关系子图聚合特征 `fA`。最后，通过对子图聚合特征进行余弦相似度交互，提取场景-音频关联特征。该过程同时利用了模态内知识和跨模态知识，有助于人格预测任务。

首先，我们提取场景图像关系子图聚合特征 `fV`。将新的场景图像特征输入两个连续的图编码器后，得到所有场景图像的特征 `f'graph^V ∈ R^{N×768}`。随后，我们使用全局最大池化提取显著特征 `fgraph^V ∈ R^{1×768}`，并通过一层全连接层得到整个场景图像关系图的最终表示 `fV ∈ R^{1×128}`：

```text
f'graph^V = σ(fgraph^V WV + bV)    (11)
fV = maxpool(f'graph^V)            (12)
```

其中，`WV` 和 `bV` 是特征映射层的权重和偏置，`maxpool` 表示全局最大池化。

与聚合场景图像关系子图特征 `fV` 的方式相同，我们也可以使用自适应图 Transformer 网络和全局最大池化来聚合音频 patch 关系子图特征 `fA`。随后，本文通过余弦相似度函数计算 `fV` 与 `fA` 的相似度分数，从而得到针对场景图像和音频 patch 的场景-音频关联特征 `fsa ∈ R^{1×128}`。原文公式（13）在 PDF 文本提取中存在排版损失，其含义为按 `128` 个维度逐项计算带有 `ε` 稳定项的余弦相似度，其中 `ε = 10^-8`，用于防止分母为 0。

#### 2.1.3 文本特征

对于文本特征，我们首先对输入视频进行预处理，得到转写文本。然后将其划分为若干句子，句子最大数量记为 `O`，少于 `O` 个句子的样本使用零向量填充。接着，将这些句子输入 CLIP 文本编码器，得到文本序列特征 `XT ∈ R^{O×768}`。在获得视频中的多个句子特征后，使用带 TAB 的 BiGRU [HZXL21] 来提取时间信息，得到文本特征 `ft`。最后，通过平均操作和 MLP，将文本特征维度压缩到 `ft ∈ R^{1×128}`。本文选择 BiGRU，是因为它在提取文本特征方面优于其他时间模型。关于不同时间模型在文本特征提取上的实验与分析，详见补充材料第 7 节。`CFI-V2` 数据集中 `O = 13`，`UDIVA v0.5` 数据集中 `O = 452`。

### 2.2 多模态通道注意力残差融合模块

现有融合方法通常直接拼接多种特征，而忽视了不同特征对人格预测的影响程度并不相同。因此，本文使用一种多模态通道注意力残差融合模块来获得三模态特征。具体来说，先将获得的三个特征向量拼接成一个融合特征向量，再将其输入该模块，以评估各分支的贡献，并尽量减少由特征多样性带来的信息冗余。同时，模块中引入残差结构，以缓解信息丢失问题。

该模块由两个全连接层组成，用于计算多模态表示 `F` 每个维度上的注意力权重 `α`：

```text
α = tanh(W2 tanh(W1 F + b) + c)    (14)
```

其中，`W1`、`W2`、`b`、`c` 分别表示两层全连接层的权重矩阵和偏置。`tanh` 将注意力权重限制在 `[-1, 1]` 区间内。随后，得到的注意力向量 `α` 与多模态向量 `F` 的各维度逐元素相乘，得到：

```text
F' = F · α + F    (15)
```

其中，`F'` 为该模块输出，它将被送入回归头以预测五个人格特质。

> **图 3：** 多模态与单模态联合学习回归头结构。MLP 模型由一个线性层和一个 sigmoid 激活层组成，将融合特征映射到 5 个维度，并将输出归一化到 0 到 1 之间，表示大五人格的预测值。

### 2.3 多任务联合学习回归头

我们主要引入一个多任务联合学习回归头来预测五个人格特质的得分。这里，三个单独的单模态任务（`fsd`、`fsa`、`ft`）与多模态任务共享模态表示，如图 3 所示。该回归头先采用多模态通道注意力残差融合模块获取融合特征，然后通过单层 MLP 模型来预测人格特质分数。

为了获得更稳健的人格预测精度，本文使用真实的大五人格标签对多模态预测头输出进行监督学习。同时，通过为每个单模态特征增加辅助预测头，实现辅助监督学习。具体来说，每个单模态分支都添加一个 MLP 模型，用于完成该单模态的人格特质预测。所有单模态分支上的 MLP 均由线性层和 sigmoid 激活层组成，输出维度为 5。需要说明的是，这些单模态辅助预测头仅用于辅助模型训练。最终，我们联合学习多模态任务和单模态任务。需要特别指出的是，这些单模态任务只在训练阶段存在，用于提升模型的鲁棒性和准确性。

### 2.4 多任务损失模块

我们通过多任务损失函数对整个框架进行联合优化，其原文定义如下：

```text
a = LossLogCosh + LossBell
LossTotal = (1/N) * Σi (a + α * Σs a)    (16)
```

其中，`LossLogCosh` 是 `logCosh loss`，`LossBell` 是 `Bell Loss`。`N` 为训练样本数量。`α` 表示单模态辅助任务 `s` 对多模态任务损失的重要性。在本文中，`α = 0.2`，`s = {fsd, fsa, ft}` 分别代表不同的单模态辅助任务。由于原 PDF 文本提取时公式排版存在轻微损失，上式按原文符号含义保留。

## 3. 实验

#### 3.3.2 自适应图 Transformer 网络的有效性

现有时间模型，如 RNN，通常面临梯度消失 / 爆炸以及长程依赖建模能力有限等问题。而本文设计的自适应图 Transformer 网络不受帧之间顺序邻接关系的限制，它能够利用完全图的拓扑结构传递和聚合特征，从而捕捉不同帧之间的长距离依赖。根据表 2 的实验结果，本文的自适应图 Transformer 网络用于挖掘两种双模态关联特征，在两个数据集的大多数指标上都优于其他时间模型，表现出较强的有效性。

#### 3.3.3 多任务损失函数的有效性

为了验证多任务损失的有效性，作者进一步比较了在不同损失函数下训练本框架时的性能，包括 `Log-Cosh` 损失、`Bell loss`、二者组合以及本文的多任务损失。该比较中，框架结构保持不变，仅替换损失函数。如表 3 所示，当同时使用两种损失时，性能有所提升，说明这两类损失具有互补性。进一步地，结合多任务损失的框架相比仅使用两种损失的组合又取得了显著改进，这验证了本文多任务损失在多模态人格分析中的有效性。

## 4. 结论

本文提出了一种新的多模态特征学习框架用于人格预测，其中包括场景-音频关联特征、场景-描述关联特征，以及带有多任务损失的多任务联合学习回归头，从而有效提升人格预测性能。大量实验验证了该框架的有效性与可靠性，结果表明本文方法优于当前最先进方法。未来工作中，我们计划探索自监督学习方法和心理学线索，以进一步提升框架性能，并将其应用于求职面试等真实场景。
