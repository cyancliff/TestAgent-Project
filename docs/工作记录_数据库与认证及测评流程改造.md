### 5. 新增最终批量提交接口

#### `POST /api/v1/assessment/submit-batch`

### 作用
用户全部题目答完后，点击“提交答卷”时：
- 一次性正式写入数据库
- 只有这一步完成后，答题数据才算有效

### 请求体结构
```json
{
  "session_id": 123,
  "answers": [
    {
      "exam_no": "DX01",
      "selected_option": "A选项内容",
      "time_spent": 1.5,
      "user_explanation": null
    }
  ]
}
```

### 后端逻辑
- 校验会话属于当前用户
- 若 session 已完成则拒绝重复提交
- 校验答案数量是否完整（必须达到 `TOTAL_QUESTIONS`）
- 校验题目编号是否合法
- 清理该 session 旧正式记录：
  - `answer_records`
  - `module_debate_results`
- 对每题：
  - 后端重新计算分数
  - 后端重新执行异常检测
  - 写入 `AnswerRecord`
- 更新 `assessment_sessions`：
  - `status='completed'`
  - `finished_at=now()`
- 自动触发已完成模块的模块辩论

### 返回
```json
{
  "status": "success",
  "message": "答题记录已正式提交",
  "saved_count": 42,
  "session_id": 123
}
```

## 十二（续）、自适应选题流程优化与Decimal错误修复

### 问题背景
用户报告测评过程中“答题之后点下一题不会给我出新的题目”，经分析发现：
1. 新测评流程改为前端本地暂存答案、最后统一 `submit-batch` 落库
2. 但 `/assessment/adaptive-question` 仍依赖数据库 `AnswerRecord` 计算已答题数和能力画像
3. 导致答题过程中后端看不到前端本地已答状态，出现下一题不推进或重复选题

### 核心目标
让自适应选题在正式提交前就基于前端本地暂存实时推进，同时保持 `submit-batch`、报告生成、历史记录等正式数据链路不变。

### 主要改动

#### 1. 扩展自适应选题请求体，显式传递前端本地已答快照
**修改文件：** `app/api/assessment.py`
- 新增 `AdaptiveAnswerItem` 类，包含：
  - `exam_no`, `selected_option`, `time_spent`, `score`, `status`, `user_explanation`
- 扩展 `AdaptiveQuestionRequest`，新增 `answers: list[AdaptiveAnswerItem]` 字段，默认空列表
- 新前端可传本地暂存快照，旧前端不传时仍可回退旧逻辑

#### 2. 在 assessment 接口层把前端答案归一化成“临时作答上下文”
**修改文件：** `app/api/assessment.py`
- 新增 `build_transient_answer_context()` 辅助函数，负责：
  - 校验 `payload.answers` 是否有重复 `exam_no`
  - 批量查询对应 `Question`
  - 生成统一的临时作答上下文：
    - `answered_question_nos`, `answered_ids`, `answered_count`
    - `question_map`, `transient_records`
- `transient_records` 字段贴近 `AnswerRecord` 的使用面，便于直接传给智能选题服务

#### 6. 修复 PostgreSQL Decimal 类型与 numpy 运算兼容问题
**问题现象：** 第三题显示智能选题，但后面就一直显示是顺序选题
**根本原因：** PostgreSQL `Numeric` 字段（`difficulty`, `discrimination`, `score`）在 SQLAlchemy 中返回 `Decimal` 类型，与 numpy 运算不兼容
**错误位置：** `app/services/question_selection.py` 第115行：`obs_precision = discrimination * 2.0 * anomaly_factor`
**修复方法：** 在所有使用 `question.difficulty`、`question.discrimination`、`record.score` 的地方显式转换为 `float`
- `get_user_ability` 函数：`difficulty = float(question.difficulty or 0.5)`
- `_select_first_question` 函数：`difficulty = float(question.difficulty or 0.5)`
- `_calculate_question_score` 函数：`difficulty = float(question.difficulty or 0.5)`
- 日志格式化也转为 `float` 避免 `Decimal` 格式化异常
