## 2026-04-08：认知诊断算法优化 - 贝叶斯能力估计 + 语义化题目参数

### 问题背景
原有自适应选题算法存在两个严重问题：
1. **题目参数无差异**：102道题中100道的难度（0.621）和区分度（0.200）完全相同，因为它们的分值结构都是 `[1,2,3,4,5]`，基于信息熵计算出的结果一模一样
2. **能力估计过于粗糙**：使用简单的平均归一化得分（`mean(score/5.0)`），所有题等权重，无法区分在难题和简单题上的表现差异

这导致选题算法中60%的权重（难度匹配30% + 区分度30%）完全失效，只有覆盖度（40%）在实际起作用。

### 解决方案

#### 1. 语义化题目参数（`generate_feature_vectors.py`）

**核心思路**：利用已有的64维语义特征向量，通过向量间的关系计算每道题独特的难度和区分度。

**难度计算**（三个因素加权）：
- **语义离群度（50%）**：题目向量离同维度中心越远 → 越不典型 → 越难理解。计算到本维度中心的距离，做z-score标准化后用sigmoid映射到0~1
- **语义模糊度（30%）**：题目与其他维度中心的最大余弦相似度。越高说明语义边界越模糊，越难判断归属
- **反向计分加成**：反向题需要逆向思维，额外+0.12难度
- 结果裁剪到 [0.10, 0.95]

**区分度计算**（两个因素加权）：
- **维度内独特性（55%）**：与同维度其他题目的平均余弦相似度越低 → 越独特 → 测到的东西越不可替代
- **维度间分离度（45%）**：与本维度中心的相似度 vs 与其他维度中心的最大相似度。差距越大 → 测量越纯净
- 结果裁剪到 [0.10, 0.95]

**更新后的参数分布**：
| 参数 | 改造前 | 改造后 |
|------|--------|--------|
| 难度 | 100题全部=0.621，2题=0.840 | 93个不同值，min=0.134, max=0.569, mean=0.353, std=0.110 |
| 区分度 | 100题全部=0.200，2题=0.500 | 55个不同值，min=0.550, max=0.950, mean=0.821, std=0.142 |

#### 2. 贝叶斯在线能力估计（`app/services/question_selection.py`）

**核心思路**：先验分布 N(0.5, 0.25)，每答一题根据得分、题目难度和区分度做正态-正态共轭更新。

**更新公式**：
```
先验精度 = 1 / σ²
观测精度 = 区分度 × 2.0 × 异常系数（异常作答降为0.3）
后验精度 = 先验精度 + 观测精度
新能力均值 = (先验精度 × 旧均值 + 观测精度 × 归一化得分) / 后验精度
新方差 = 1 / 后验精度
```

**关键特性**：
- 高区分度的题对能力判断影响更大
- 异常作答（is_anomaly=1）的观测精度降低70%，减少噪声影响
- 输出能力不确定性 σ²，可驱动选题策略

**新增兼容接口**：`get_user_ability()` 返回4元组 `(profile_vector, ability_level, ability_uncertainty, answered_vectors)`，同时保留旧的 `get_user_ability_vector()` 接口兼容性

#### 3. Fisher信息量 + 动态权重选题（`app/services/question_selection.py`）

**新增第四维度——Fisher信息量**：
```python
fisher_info = discrimination² × exp(-(difficulty - ability)² / (2 × 0.15))
```
区分度高 + 难度接近能力水平 → 该题能最有效地缩小不确定性。

**动态权重机制**（根据 `uncertainty_ratio = min(σ² / 0.25, 1.0)` 调整）：

| 维度 | 不确定性高（前几题） | 不确定性低（后面的题） |
|------|---------------------|---------------------|
| Fisher信息量 | 0.40（快速锁定能力） | 0.15 |
| 覆盖度 | 0.25 | 0.40（拓宽诊断面） |
| 难度匹配 | 0.20 | 0.25 |
| 区分度 | 0.15 | 0.20 |

**效果**：前几题优先选高区分度、难度适中的题快速锁定能力范围；后面的题优先选与已答题语义差异大的题，拓宽诊断覆盖面。
