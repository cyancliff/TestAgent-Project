> 更新基线：2026-04-18

## 1. 课题总体目标

本课题的总目标，是构建一个面向心理测评场景的智能评估系统，使系统不只依赖传统问卷分数，而是能够结合：

1. 问卷作答内容。
2. 作答过程行为信号。
3. 大模型辩论推理结果。
4. RAG 心理学知识证据。
5. 视频驱动的视觉多模态人格测评结果。

最终形成一个“**问卷测评 + 智能推理 + 多模态补充证据**”的一体化毕设系统。

围绕这个目标，系统的研究与工程产出分为三层：

- **应用层目标**：实现可用的心理测评 Web 系统，支持注册登录、答题、报告生成、历史记录和 AI 咨询。
- **算法层目标**：实现自适应选题、异常作答检测、多智能体辩论和视觉多模态人格预测。
- **论文层目标**：形成清晰的系统设计、关键算法说明、实验方案、对比分析和阶段性工程成果。

## 2. 总体技术路线

### 2.1 问卷测评主线

当前系统主线基于 ATMR 四维模型展开，完整链路如下：

```text
用户登录
-> 创建测评会话
-> 固定题 + A/T/M/R 四阶段答题
-> 自适应选题
-> 异常作答检测
-> 模块级多智能体辩论
-> 综合报告生成
-> 测评后 AI 心理咨询
```

该部分已经是当前系统的主功能，负责输出 ATMR 四维报告，是毕设的核心主体。

### 2.2 视觉多模态测评支线

新增的视觉多模态测评部分不直接替代问卷评分，而是作为一个**辅助画像子系统**，负责从用户自拍视频中提取视觉、音频和文本线索，预测 Big Five 五维人格结果，再作为补充证据接入报告。

其技术路线如下：

```text
上传视频
-> 创建多模态分析任务
-> 抽帧 / 音频提取 / 语音转写
-> 视觉与文本特征提取
-> 多模态模型推理
-> 输出 Big Five 五维结果
-> 接入测评报告 / 辩论上下文 / 辅助画像说明
```

### 2.3 双链路融合原则

视觉多模态部分目前遵循以下原则：

1. **先独立运行，再逐步融合**：先保证视频分析链路可复现、可调用、可解释，再考虑和 ATMR 深度融合。
2. **先并列展示，再做联合建模**：前期将 Big Five 结果作为“辅助画像章节”放入报告，不直接改写 ATMR 分数。
3. **先工程闭环，再追求论文指标**：优先完成数据处理、特征提取、模型接入和接口联调，再推进模型优化和实验复现。

## 3. 毕设核心开发目标

### 3.1 问卷测评系统目标

#### 目标一：完成 ATMR 智能测评主流程

- 构建基于 FastAPI + Vue 3 的完整测评系统。
- 实现用户登录、会话创建、断点续答、阶段答题、报告查看和历史记录查询。
- 建立 ATMR 四维评分体系与结构化报告页面。

#### 目标二：实现自适应选题与异常检测

- 基于题目特征向量、难度、区分度和贝叶斯能力估计实现智能选题。
- 对作答过快、过慢、连续快速作答、重复选项等异常模式进行检测。
- 将异常检测结果作为后续辩论与报告生成的重要证据。

#### 目标三：实现多智能体辩论和 RAG 支撑

- 基于 AutoGen 构建“正方专家 - 反方专家 - 裁决官”的多智能体辩论流程。
- 接入 ATMR 知识库，通过 RAG 为辩论和报告提供理论依据。
- 生成具有解释性和证据链的深度心理画像报告。

---

### 3.2 视觉多模态测评目标

视觉多模态测评是本次文档重构的新增重点。该部分以**视频为主输入**，通过视觉、音频、文本三类信息建立辅助人格预测能力。

#### 目标一：构建独立的视觉多模态分析子系统

目标不是把论文代码直接塞进主项目，而是先把多模态能力做成一个独立、稳定、可调用的子系统。该子系统应满足：

1. 可接受本地视频路径或前端上传视频。
2. 可进行预处理、特征提取、推理和结果查询。
3. 可输出标准化 Big Five 五维结果与模型版本信息。
4. 可在主系统中通过 API 接入，而不破坏现有 ATMR 主链路。

#### 目标二：完成视频驱动的多模态预处理链路

该部分当前的技术设计已经明确，目标包括：

1. **视频抽帧**  
   使用 `ffmpeg` 对输入视频进行均匀采样，当前占位版流程会抽取最多 15 张关键帧，并统一缩放到 `336x336`，用于后续视觉编码。

2. **音频提取**  
   从原视频中分离单声道 `16kHz` 的 WAV 音频，用于后续语音建模或音频特征提取。

3. **自动语音转写**  
   使用本地 Whisper 模型完成音频转写，当前占位版采用 `whisper tiny` 作为轻量起点，后续可视情况替换为更高精度模型。

4. **工件归档**  
   每次分析都应生成独立工件目录，记录：
   - 帧图像目录
   - 音频文件
   - 转写文本
   - 预处理 manifest
   - 错误信息

5. **任务状态可追踪**  
   分析任务至少支持 `pending / running / completed / failed` 四种状态，并能够返回创建时间、更新时间、错误原因和工件路径。

#### 目标三：完成视觉和文本特征提取链路

多模态测评的核心并不只是“把视频上传成功”，而是把视频转成模型可用的特征表示。当前与后续目标如下：

1. **视觉特征提取**  
   使用 CLIP 模型提取视频关键帧图像嵌入，当前计划采用 `openai/clip-vit-large-patch14-336`。

2. **文本特征提取**  
   使用 CLIP 的文本编码器对 Whisper 转写文本进行编码，形成与视觉特征同空间或可对齐的语义特征。

3. **音频特征提取**  
   当前仓库还未完成真实音频特征提取，后续目标是接入 `wav2clip` 或等价音频编码方案，将说话节奏、语音能量和声学线索纳入模型输入。

4. **多模态样本对齐**  
   需要建立统一的数据样本结构，使同一个视频样本同时包含：
   - 视频路径
   - 抽帧结果
   - 音频路径
   - 转写文本
   - 标签（Big Five）
   - 特征文件路径

#### 目标四：完成论文模型复现与 Big Five 推理

视觉多模态部分的学术目标，不是直接输出 ATMR 四维，而是先复现或近似复现论文中的 **Big Five 人格预测**。当前规划如下：

1. **短期目标**  
   先完成“占位版推理链路”，即：
   - 预处理成功
   - 特征文件生成成功
   - 接口返回五维结果结构
   - 可被主系统读取

2. **中期目标**  
   参考论文《A multimodal personality prediction framework based on adaptive graph transformer network and multi-task learning》，逐步补齐：
   - 多模态输入拼装
   - 自适应图结构建模
   - 多任务学习目标
   - Big Five 五维联合预测

3. **长期目标**  
   在模型稳定后，将 Big Five 结果转换为：
   - 测评报告的辅助画像段落
   - 多智能体辩论的额外证据输入
   - 问卷结果一致性分析材料

#### 目标五：实现视觉多模态结果与 ATMR 报告的渐进融合

这一部分是毕设的关键亮点之一，但必须分阶段推进：

1. **第一阶段：并列展示**  
   在测评报告中新增“视觉多模态人格分析”章节，展示 Big Five 五维分数和解释文字。

2. **第二阶段：辅助解释**  
   将多模态结果作为多智能体辩论的辅助上下文，例如：视频表现出的外向性、尽责性是否与问卷作答一致。

3. **第三阶段：一致性分析**  
   研究 Big Five 与 ATMR 维度之间的可能映射关系，用于论文讨论和误差分析，但不直接进行硬性的数值融合。

## 4. 视觉多模态测评的详细技术方案

### 4.1 当前代码实现基础

当前仓库已经为视觉多模态测评搭建了独立骨架，主要包括：

- 接口层：`app/api/multimodal_personality.py`
- 服务层：`app/services/multimodal_personality_service.py`
- 数据结构：`app/schemas/multimodal_personality.py`
- 推理流水线：`multimodal_personality/inference/pipeline.py`
- 数据预处理：`multimodal_personality/preprocessing/cfi_v2_dataset.py`
- 特征提取：`multimodal_personality/feature_extractors/clip_extractor.py`
- 数据脚本：
  - `scripts/build_cfi_v2_manifest.py`
  - `scripts/preprocess_cfi_v2_from_manifest.py`
  - `scripts/build_clip_feature_jobs.py`
  - `scripts/extract_clip_features.py`

### 4.2 当前多模态接口能力

后端目前已经支持以下接口：

- `POST /api/v1/multimodal-personality/upload`
- `POST /api/v1/multimodal-personality/upload-file`
- `POST /api/v1/multimodal-personality/run`
- `GET /api/v1/multimodal-personality/result/{task_id}`
- `GET /api/v1/multimodal-personality/health`

这些接口已经可以完成：

1. 创建视频分析任务。
2. 保存上传视频。
3. 触发预处理流水线。
4. 查询任务状态与工件路径。
5. 暴露系统工具依赖检查结果。

### 4.3 当前占位版推理流水线细节

当前版本还不是完整论文模型，而是“可运行的工程骨架”。其实际流程为：

1. 检查 `ffmpeg`、`ffprobe`、`whisper` 可用性。
2. 对输入视频抽帧。
3. 从视频中提取音频。
4. 调用 Whisper 做本地转写。
5. 生成 `manifest.json` 记录阶段执行状态。
6. 若预处理成功，则返回占位版 Big Five 五维结果。

当前占位版模型版本为：

- `scaffold-v1`

当前占位版输出的五维结果是中性占位值，作用是：

1. 保证接口结构提前稳定。
2. 保证主系统可以先联调。
3. 为后续替换为真实模型推理做好边界隔离。

### 4.4 数据集与样本组织目标

当前多模态模块以 `CFI-V2` 为核心样本组织对象，计划实现以下数据工程目标：

1. 将训练集、验证集、测试集统一配置到 YAML 中。
2. 从 `annotation.pkl` 和 `transcription.pkl` 中解析出：
   - 样本视频名
   - Big Five 标签
   - 转写文本
3. 结合视频目录生成标准化 `manifest` 文件。
4. 统计：
   - 样本数量
   - 缺失视频数量
   - 缺失转写数量

这一层的目标不是模型本身，而是保证数据可追踪、可重跑、可复现实验。

### 4.5 视觉多模态子系统的阶段验收目标

#### 阶段 A：工程骨架验收

- 能上传或登记视频任务。
- 能产出工件目录和阶段 manifest。
- 能通过 API 查询状态。

#### 阶段 B：预处理链路验收

- 视频抽帧成功。
- 音频提取成功。
- Whisper 转写成功。
- 失败时能写出明确错误，而不是静默失败。

#### 阶段 C：特征提取链路验收

- 可从 manifest 构造 CLIP 特征任务。
- 可输出图像特征和文本特征 JSON。
- 缺依赖或缺帧时可精确报错。

#### 阶段 D：真实推理链路验收

- 可加载真实 checkpoint。
- 可完成 Big Five 五维前向推理。
- 可返回 `scores + model_version + artifacts + status`。

#### 阶段 E：主系统接入验收

- 报告中新增多模态章节。
- 可将结果挂载到测评会话上下文。
- 可为后续多智能体辩论提供辅助证据。

## 7. 实验与论文写作目标

- 对比传统固定选题与自适应选题在效率和信息量上的差异。

- 验证视频预处理链路对真实样本的可用性。
- 验证 CLIP 视觉/文本特征提取的稳定性。
- 在 CFI-V2 数据上完成 Big Five 五维预测实验。
- 对比单模态与多模态方案的效果差异：
  - 仅视觉
  - 仅文本
  - 视觉 + 文本
  - 视觉 + 文本 + 音频

### 7.3 融合分析目标

- 分析问卷结果与视频预测结果的一致与不一致样本。
- 探索 Big Five 与 ATMR 之间的解释映射关系。
- 将融合结果作为论文创新点之一：不仅做问卷测评，还引入了外显行为证据。

## 9. 文档结论

当前毕设已经不再是单纯的“心理问卷网站”，而是一个逐步成型的智能心理测评平台：

- 主体部分是 ATMR 问卷测评、异常检测、多智能体辩论与 RAG 报告。
- 新增亮点部分是视觉多模态测评，它将视频中的视觉、语音和文本线索引入人格预测。

从毕设写作角度看，后续最有价值的创新表达是：

1. **从传统静态问卷走向智能交互测评。**
2. **从单一作答结果走向“作答行为 + 大模型推理 + 知识证据”的综合分析。**
3. **从单模态问卷系统扩展到“问卷 + 视频”的视觉多模态测评系统。**

这也是后续系统实现、答辩汇报和论文撰写时最应该突出的主线。
