# ATMR 心理测评系统 - 代码实况整理

> 本文档基于当前代码库实际实现情况整理，不以历史任务文档为依据。

## 当前系统状态

### 已确认已实现
- **后端框架**：FastAPI，入口为 `app/main.py`
- **前端框架**：Vue 3 + Vite，位于 `frontend/src/`
- **用户认证**：已实现注册、登录、JWT 鉴权、密码加密兼容
- **测评流程**：已实现 42 题完整测评流程（前 2 题固定 + ATMR 四模块各 10 题）
- **智能选题**：已实现贝叶斯能力估计、Fisher 信息量、动态权重选题
- **RAG 系统**：已接入辩论、报告、聊天，基于 PageIndex 知识库检索
- **多智能体辩论**：已实现正方 / 反方 / 裁决三角色辩论
- **前端交互**：已实现断点续答、修改答案重提、题目导航、报告展示、聊天页面
- **部署配置**：已具备 Docker Compose、Nginx、前后端容器化配置

### 当前代码层面可见的主要短板
- `app/api/assessment.py` 存在重复端点定义
- `app/api/chat.py` 的聊天历史仅保存在内存中
- `app/services/ai_detector.py` 的异常检测逻辑较弱
- 数据库虽已支持 PostgreSQL，但迁移与结构优化尚未真正落地
- 尚未看到 Redis、Celery、Alembic、asyncpg 的实际集成代码
