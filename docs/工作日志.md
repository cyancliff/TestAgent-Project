## 2026-04-08：认知诊断算法优化 - 贝叶斯能力估计 + 语义化题目参数

### 问题背景
原有自适应选题算法存在两个严重问题：
1. **题目参数无差异**：102道题中100道的难度（0.621）和区分度（0.200）完全相同，因为它们的分值结构都是 `[1,2,3,4,5]`，基于信息熵计算出的结果一模一样
2. **能力估计过于粗糙**：使用简单的平均归一化得分（`mean(score/5.0)`），所有题等权重，无法区分在难题和简单题上的表现差异

这导致选题算法中60%的权重（难度匹配30% + 区分度30%）完全失效，只有覆盖度（40%）在实际起作用。

### 解决方案

#### 1. 语义化题目参数（`generate_feature_vectors.py`）

**核心思路**：利用已有的64维语义特征向量，通过向量间的关系计算每道题独特的难度和区分度。

**难度计算**（三个因素加权）：
- **语义离群度（50%）**：题目向量离同维度中心越远 → 越不典型 → 越难理解。计算到本维度中心的距离，做z-score标准化后用sigmoid映射到0~1
- **语义模糊度（30%）**：题目与其他维度中心的最大余弦相似度。越高说明语义边界越模糊，越难判断归属
- **反向计分加成**：反向题需要逆向思维，额外+0.12难度
- 结果裁剪到 [0.10, 0.95]

**区分度计算**（两个因素加权）：
- **维度内独特性（55%）**：与同维度其他题目的平均余弦相似度越低 → 越独特 → 测到的东西越不可替代
- **维度间分离度（45%）**：与本维度中心的相似度 vs 与其他维度中心的最大相似度。差距越大 → 测量越纯净
- 结果裁剪到 [0.10, 0.95]

**更新后的参数分布**：
| 参数 | 改造前 | 改造后 |
|------|--------|--------|
| 难度 | 100题全部=0.621，2题=0.840 | 93个不同值，min=0.134, max=0.569, mean=0.353, std=0.110 |
| 区分度 | 100题全部=0.200，2题=0.500 | 55个不同值，min=0.550, max=0.950, mean=0.821, std=0.142 |

#### 2. 贝叶斯在线能力估计（`app/services/question_selection.py`）

**核心思路**：先验分布 N(0.5, 0.25)，每答一题根据得分、题目难度和区分度做正态-正态共轭更新。

**更新公式**：
```
先验精度 = 1 / σ²
观测精度 = 区分度 × 2.0 × 异常系数（异常作答降为0.3）
后验精度 = 先验精度 + 观测精度
新能力均值 = (先验精度 × 旧均值 + 观测精度 × 归一化得分) / 后验精度
新方差 = 1 / 后验精度
```

**关键特性**：
- 高区分度的题对能力判断影响更大
- 异常作答（is_anomaly=1）的观测精度降低70%，减少噪声影响
- 输出能力不确定性 σ²，可驱动选题策略

**新增兼容接口**：`get_user_ability()` 返回4元组 `(profile_vector, ability_level, ability_uncertainty, answered_vectors)`，同时保留旧的 `get_user_ability_vector()` 接口兼容性

#### 3. Fisher信息量 + 动态权重选题（`app/services/question_selection.py`）

**新增第四维度——Fisher信息量**：
```python
fisher_info = discrimination² × exp(-(difficulty - ability)² / (2 × 0.15))
```
区分度高 + 难度接近能力水平 → 该题能最有效地缩小不确定性。

**动态权重机制**（根据 `uncertainty_ratio = min(σ² / 0.25, 1.0)` 调整）：

| 维度 | 不确定性高（前几题） | 不确定性低（后面的题） |
|------|---------------------|---------------------|
| Fisher信息量 | 0.40（快速锁定能力） | 0.15 |
| 覆盖度 | 0.25 | 0.40（拓宽诊断面） |
| 难度匹配 | 0.20 | 0.25 |
| 区分度 | 0.15 | 0.20 |

**效果**：前几题优先选高区分度、难度适中的题快速锁定能力范围；后面的题优先选与已答题语义差异大的题，拓宽诊断覆盖面。

## 2026-04-11：整体前端梳理与统一优化

### 问题分析
经过完整阅读全部前端核心文件后发现：
- 全局样式存在双源（`style.css` 与 `App.vue`），且 `style.css` 含大量 Vite 模板残留
- 页面骨架不统一（各页 max-width、padding、首屏间距各不相同）
- History 和 Chat 的侧栏样式各自实现，命名和细节不一致
- 按钮体系碎片化（各页自定义 `.btn-view`、`.btn-edit`、`.btn-chat`、`.btn-delete`、`.btn-icon`、`.start-btn` 等）
- 加载/空状态/错误反馈方式不统一

### 修复内容

#### 1. 收口全局样式来源
**`frontend/src/style.css`**：
- 清掉全部 Vite 模板残留样式（`.hero`、`#center`、`#next-steps`、`#spacer`、`.ticks`、`.counter`、嵌套 CSS 写法等）
- 只保留基础 reset（`*`、`body`）和 `#app` 最小容器

**`frontend/src/App.vue`**：
- 成为唯一的全局样式入口
- 承载：设计变量、全局 reset、导航栏、页面壳层、通用组件样式

#### 2. 建立统一页面壳层（App.vue 新增）
| 类名 | 用途 |
|------|------|
| `.page-layout` | 左栏 280px + 右栏 grid 二栏布局 |
| `.page-layout--no-sidebar` | 无侧栏的单栏页面 |
| `.page-sidebar` | 统一侧栏容器（sticky、圆角、阴影、max-height） |
| `.sidebar-header` / `.sidebar-list` / `.sidebar-item` | 侧栏内部统一结构 |
| `.sidebar-item.active` | 侧栏激活项 |
| `.sidebar-actions` | 侧栏操作区 |
| `.page-header` / `.page-title` / `.page-subtitle` | 统一页面头部 |
| `.state-block` / `.spinner` | 统一加载/空状态块 |
| `.btn-danger` | 统一危险操作按钮 |
| `.btn-ghost` | 统一次级/文本按钮 |

含 `@media (max-width: 1200px)` 自动回退单栏。

#### 3. 重写 Chat.vue
- 模板使用统一的 `page-layout` + `page-sidebar` + `sidebar-*` 类
- 侧栏用全局样式，页面内只保留聊天特有样式
- 发送消息后优先采用后端返回的完整 `messages` 数组同步
- 清掉之前多轮修补遗留的重复样式和断裂 DOM 层级
- 气泡、输入区、Markdown 样式全部重写，与全局 token 一致

#### 4. 统一 History.vue
- 侧栏改用全局 `page-layout` + `page-sidebar` + `sidebar-*`
- 删除页面内重复的侧栏样式定义（约 60 行）
- 修正模板缩进问题

#### 5. 对齐 Assessment.vue 和 Report.vue
- Assessment 去掉硬编码的 `max-width: 2100px`，让外层壳层控制宽度
- Report 顶部间距从 `48px` 统一到 `32px`，去掉多余 padding
