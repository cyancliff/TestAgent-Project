## 1. 系统概述

### 1.1 项目背景

TestAgent 是一个基于多智能体辩论（Multi-Agent Debate）和自适应题目选择（Computerized Adaptive Testing, CAT）的 AI 心理测评系统。系统融合贝叶斯能力估计、异常作答检测、RAG 知识库检索和大语言模型推理，实现对受试者 **态度(A)**、**情绪(T)**、**认知(M)**、**韧性(R)** 四个维度的智能化心理评估。

### 1.2 系统定位

- **测评对象**：需要心理画像分析的个体用户
- **测评维度**：ATMR 四维特质模型（欣赏型 Appreciation / 目标型 Target / 包容型 Magnanimity / 责任型 Responsibility）
- **输出产物**：雷达图评分报告 + 多智能体辩论生成的深度心理画像 + 可持续对话的 AI 心理咨询

## 4. 核心算法实现

### 4.1 自适应题目选择算法

自适应选题是系统的核心创新点之一，采用贝叶斯能力估计与 Fisher 信息量相结合的动态选题策略。

#### 4.1.1 贝叶斯能力估计

**先验分布**：假设用户能力 θ 服从正态分布 N(μ=0.5, σ²=0.25)，即初始能力估计为中等水平，不确定度为 0.25。

**共轭更新**：每答一题后，根据作答得分和题目参数进行贝叶斯后验更新：

```
先验精度 = 1 / σ²
观测精度 = 区分度 × 2.0 × 异常因子
后验精度 = 先验精度 + 观测精度
后验均值 = (先验精度 × 先验均值 + 观测精度 × 得分归一化值) / 后验精度
后验方差 = 1 / 后验精度
```

其中，异常作答（作答过快/过慢/重复选项）的异常因子为 0.3，正常作答为 1.0，即异常作答的权重降低，防止噪声数据污染能力估计。

#### 4.1.2 用户画像向量

用户画像向量是所有已答题目特征向量的加权平均：

```
profile = Σ(wi × vi) / Σwi
```

其中 vi 为第 i 题的特征向量，wi 为权重（异常作答 wi=0.5，正常作答 wi=1.0）。特征向量由 Sentence-Transformers 生成并经 PCA 降维至 64 维，编码题目的语义特征。

#### 4.1.3 动态权重选题策略

候选题目的推荐分数由四个维度加权计算，权重随能力不确定性动态调整：

| 维度 | 计算方式 | 权重范围 | 作用 |
|------|----------|----------|------|
| Fisher 信息量 | a² × exp(-(b-θ)²/0.3) | 0.15 ~ 0.40 | 快速锁定用户能力 |
| 覆盖度 | 1 - 平均余弦相似度 | 0.25 ~ 0.40 | 拓宽诊断面 |
| 难度匹配 | 1 - |b - θ| | 0.20 ~ 0.25 | 匹配用户能力水平 |
| 区分度 | a（题目区分度） | 0.15 ~ 0.20 | 选择高区分度题目 |

其中 a 为区分度参数，b 为难度参数，θ 为用户能力水平。不确定性高时（答题初期）更看重 Fisher 信息量和区分度以快速收敛能力估计；不确定性低时（答题后期）更看重覆盖度以拓宽诊断维度。

#### 4.1.4 首题选择策略

首次答题时无用户画像，采用简化策略：选择中等难度（difficulty≈0.5）+ 高区分度的题目，快速建立能力基线。

### 4.2 异常作答检测算法

实时检测以下异常模式：

| 检测维度 | 阈值 | 风险分 | 追问话术 |
|----------|------|--------|----------|
| 作答时间过快 | < 20% 预估时间 | +70 | 你这题答得很快，能具体说说为什么这样选吗？ |
| 作答时间偏快 | < 35% 预估时间 | +45 | 同上 |
| 作答时间过长 | > max(3倍预估, 60秒) | +20 | 你思考了比较久，当时在犹豫什么？ |
| 连续快速作答 | 最近 4 题中 ≥3 题快速 | +30 | 同上 |
| 重复选项 | 最近 3 题选择相同选项 | +25 | 你最近选择模式比较一致，说说如何判断的？ |

风险分 ≥45 触发异常标记，系统自动生成追问，要求用户补充说明。用户的解释将被记录并与作答数据一并送入辩论分析环节。

### 4.3 多智能体辩论系统

#### 4.3.1 三方辩论架构

基于 Microsoft AutoGen 的 GroupChat 框架，构建三个 AI 智能体：

| 智能体 | 角色 | 模型 | 分析视角 |
|--------|------|------|----------|
| 正方心理学专家 | Proponent | DeepSeek-Chat | 积极角度分析用户潜力、自我效能感、抗压能力 |
| 反方风控专家 | Opponent | 通义千问 Qwen | 批判性审查异常作答、潜在性格弱点和风险 |
| 主裁决官 | Judge | 智谱 GLM-4 | 控场辩论、综合双方论点、输出最终评估报告 |

#### 4.3.2 辩论流程

1. **数据注入**：将用户作答记录（含异常检测和用户解释）组装为 JSON 上下文
2. **RAG 增强**：从 ATMR 知识库检索相关理论作为辩论依据
3. **多轮辩论**：正方 → 反方 → 裁决官轮流发言（最多 7 轮）
4. **报告提取**：从裁决官的最终发言中提取结构化心理画像报告

#### 4.3.3 模块辩论与综合报告

系统支持两级辩论：

- **模块级辩论**：每完成一个维度模块（10 题）后，由三位专家（心理学专家、行为分析师、批判性评估师）并行分析该模块数据，生成模块评估结论
- **综合报告**：所有模块完成后，由 AutoGen 三方辩论引擎基于全部作答记录和模块辩论结果生成最终深度报告

#### 4.3.4 SSE 实时推送

辩论过程通过 Server-Sent Events (SSE) 实时推送到前端，用户可查看各智能体的分析过程，缓解等待焦虑。辩论完成后，报告自动保存至数据库。

### 4.4 RAG 知识库系统

#### 4.4.1 知识库架构

基于 PageIndex 检索引擎，对 ATMR 心理学理论文档进行语义索引：

1. **文档解析**：将 ATMR 心理学文档通过 MinerU 转换为 Markdown 格式
2. **索引构建**：使用 DeepSeek API 构建文档的层次化语义索引（目录树结构）
3. **两阶段检索**：
   - **阶段一**：根据查询关键词从目录树中匹配相关章节
   - **阶段二**：从匹配章节中提取正文内容
4. **关键词映射**：建立 ATMR 专业术语的同义词映射（如"A特质"→"欣赏型/appreciation"）

#### 4.4.2 RAG 应用场景

| 场景 | 检索策略 | 注入方式 |
|------|----------|----------|
| 辩论增强 | 根据用户特质分数检索相关理论 | 辩论 prompt 中注入 ATMR 理论依据 |
| 报告生成 | 综合各维度得分检索对应理论 | 报告上下文中附加知识库引用 |
| 咨询对话 | 根据用户提问实时检索 | 对话消息中附加相关知识片段 |

## 9. 关键技术决策

### 9.1 为什么选择 AutoGen 而非 LangChain

AutoGen 提供了原生的多智能体对话编排能力，通过 GroupChat 和 GroupChatManager 可直接实现正方-反方-裁决官的三方辩论流程，无需手动维护对话状态和消息路由。相比之下，LangChain 更偏向单 Agent 的 Chain/Tool 范式，实现多智能体辩论需要大量自定义代码。

### 9.2 为什么使用贝叶斯方法而非 IRT

经典 IRT（项目反应理论）需要大量样本进行参数标定，不适合毕设场景的小样本需求。贝叶斯能力估计可以在少量作答（10 题/模块）的情况下给出合理的能力后验分布，且与 Fisher 信息量结合可实现动态选题，更符合计算机化自适应测评（CAT）的实践。

### 9.3 为什么采用多模型混合策略

系统同时接入 DeepSeek（正方）、通义千问（反方）、智谱 AI（裁决官）三个不同模型，旨在：
1. 利用不同模型的知识分布差异，增强辩论的多样性
2. 避免单一模型的系统性偏见影响评估结果
3. 提高系统的容错能力，单一模型不可用时可降级使用其他模型

### 9.4 RAG vs 微调

系统采用 RAG（检索增强生成）而非模型微调方案，原因是：
1. ATMR 心理学知识库是结构化文档，适合检索增强而非微调
2. RAG 提供了可追溯的证据链，便于评估结果的可解释性
3. 知识库更新无需重新训练模型，维护成本低

### 9.5 SSE vs WebSocket

选择 SSE（Server-Sent Events）而非 WebSocket 实现辩论流推送，原因是：
1. 辩论流是单向的（服务器 → 客户端），SSE 语义更匹配
2. SSE 基于 HTTP，无需额外协议支持，Nginx 配置简单
3. 自动重连和心跳机制由浏览器原生支持
